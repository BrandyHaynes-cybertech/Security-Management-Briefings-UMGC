# Security Briefing — Software Development Security & CMMI Level 3
**Course:** CMIT 425 — Cybersecurity Management | UMGC  
**Author:** Brandy Haynes  

---

## Purpose

Developing a standard set of security review procedures for virtual teams ensures that software security compliance requirements are being met and verified. Doing so positions the company to achieve **CMMI Level 3** compliance.

---

## CMMI Level 3 Defined

**Capability Maturity Model Integration (CMMI) Level 3** is characterized by:
- A clearly defined set of frameworks with standardized practices across all areas and projects
- Projects that can be tailored to each project's specific needs
- Benefits including:
  - Enhanced process predictability
  - Improved quality
  - Security integrated into the development lifecycle

---

## Software Development Security Fundamentals

### Why Security Matters in Development
Security is a critical component of software development to ensure privacy, data protection, and user trust.

### Challenges That Complicate Security
- Rapidly evolving environments
- Increased market expectations
- Rapid technology changes
- Outdated software or hardware

---

## Establishing Security Requirements

Security requirements must be defined at the **start of the development lifecycle** using:

- **Documentation tools** to track security requirements throughout development:
  - JIRA, Confluence, Trello, Asana, GitHub

- **Stakeholder Engagement** to:
  - Ensure all parties are involved in the process
  - Increase likelihood of future buy-in and support
  - Improve risk management through diverse insights

---

## Integrating Security Into Development

### Secure Coding Practices
- Adopt standards like the **OWASP Secure Coding Guidelines**
- Identify potential threats and vulnerabilities as early as possible

### Automated Security Testing Tools
- SonarQube
- Snyk
- Veracode
- Fortify

### Regular Assessments
- Conduct periodic security audits
- Report compliance efforts to stakeholders
- Maintain thorough documentation for security and audit purposes

### Industry Standards Compliance
- **OWASP** — Open Web Application Security Project
- **ISO/IEC** — International security standards
- **NIST** — National Institute of Standards and Technology

---

## Virtual Team Security Review Processes

### Communication Tools for Collaboration
- Slack
- Microsoft Teams
- Google Workspace
- Zoom

### Clear Role Definition
Define roles, responsibilities, authorization levels, and authentication requirements for all security-related tasks.

### Remote Security Review Procedures
- Conduct security reviews via virtual and in-person meetings
- Ensure audits are conducted appropriately and consistently
- Maintain shared documentation accessible to all authorized team members

---

## Continuous Improvement

- **Periodic training** — stay compliant with the latest security standards
- **Utilize feedback** from security reviews and personnel to improve network security concerns
- **Stay current** through webinars, specialized courses, and technical blogs
